# Usage

DisMusicPresence does not have user-facing runtime commands yet. Version `0.0.1` establishes project documentation, licensing, ignore rules, and maintainer workflow.

## Planned Usage Model

The application is expected to run locally and update Discord presence based on the active media source.

Planned examples:

- Apple Music track playback can publish a listening presence.
- Plex playback discovered through Tautulli can publish a watching presence.
- Formatting can be adjusted through local settings once configuration support exists.

Platform support will be developed primarily on macOS. Windows and Linux compatibility are goals, but early development will prioritize the maintainer's macOS workflow.
